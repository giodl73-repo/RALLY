# RALLY Waves

Work is organized into famous speed-race themed phases and small pulses. The
names are planning mnemonics for repeatable runs, validation pressure, and
adapter endurance; they are not endorsements of unsafe driving or illegal
racing.

## Phase map

| Phase | Name | Focus | Status |
|---:|---|---|---|
| 1 | Cannonball Run | Foundation, core crate, docs, and first adoption plans | done |
| 2 | Mille Miglia | AMAZE adapter extraction from the existing Rust harness | done |
| 3 | Dakar Rally | QUEST full Rust mechanics conversion backed by RALLY seeds | done |
| 4 | 24 Hours of Le Mans | HUNT and TIGRIS endurance validation adapters | active |

## Active wave

- 24 Hours of Le Mans: HUNT/TIGRIS simulator pilots are underway.

## Protocol

1. Read this file.
2. Read the active wave `WAVE.md`.
3. Read the target pulse under `pulses/`.
4. Implement the smallest complete slice.
5. Keep product-specific behavior out of shared kernels.
6. Update docs and wave/pulse status.
7. Run the repo validation commands.
8. Commit when green.
